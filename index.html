使用 Shared Memory 优化矩阵乘法

在 CUDA 中，全局内存的访问延迟远高于共享内存。通过将数据分块加载到 shared memory，可以显著减少全局内存访问次数，从而大幅提升矩阵乘法的性能。

__global__ void matMulShared(float *A, float *B, float *C, int N) {
    __shared__ float tileA[TILE_SIZE][TILE_SIZE];
    __shared__ float tileB[TILE_SIZE][TILE_SIZE];

    int row = blockIdx.y * TILE_SIZE + threadIdx.y;
    int col = blockIdx.x * TILE_SIZE + threadIdx.x;
    float sum = 0.0f;

    for (int t = 0; t < N / TILE_SIZE; t++) {
        tileA[threadIdx.y][threadIdx.x] = A[row * N + t * TILE_SIZE + threadIdx.x];
        tileB[threadIdx.y][threadIdx.x] = B[(t * TILE_SIZE + threadIdx.y) * N + col];
        __syncthreads();

        for (int k = 0; k < TILE_SIZE; k++)
            sum += tileA[threadIdx.y][k] * tileB[k][threadIdx.x];
        __syncthreads();
    }
    C[row * N + col] = sum;
}
Tip: TILE_SIZE 通常取 16 或 32。保持 tile 大小与 warp size (32) 对齐可以最大化内存合并访问效率。
CUDA 2026-02-18

CUDA Stream 实现计算与传输重叠

默认情况下 CUDA 操作是串行的。使用多个 stream 可以让数据传输和 kernel 执行并行进行，有效隐藏传输延迟。

cudaStream_t stream1, stream2;
cudaStreamCreate(&stream1);
cudaStreamCreate(&stream2);

// stream1: 传输前半部分数据并计算
cudaMemcpyAsync(d_a, h_a, halfSize, cudaMemcpyHostToDevice, stream1);
kernel<<<gridHalf, block, 0, stream1>>>(d_a, d_out);

// stream2: 传输后半部分数据并计算
cudaMemcpyAsync(d_a + half, h_a + half, halfSize, cudaMemcpyHostToDevice, stream2);
kernel<<<gridHalf, block, 0, stream2>>>(d_a + half, d_out + half);

cudaStreamSynchronize(stream1);
cudaStreamSynchronize(stream2);
Tip: 使用 cudaMallocHost 分配 pinned memory 才能真正实现异步传输，普通 malloc 会退化为同步操作。

用 PyTorch 实现自定义学习率调度器

内置的调度器不能满足所有需求。通过继承 _LRScheduler，可以轻松实现 warmup + cosine decay 等复合策略。

import math
from torch.optim.lr_scheduler import _LRScheduler

class WarmupCosineScheduler(_LRScheduler):
    def __init__(self, optimizer, warmup_steps, total_steps, min_lr=1e-6):
        self.warmup_steps = warmup_steps
        self.total_steps = total_steps
        self.min_lr = min_lr
        super().__init__(optimizer)

    def get_lr(self):
        step = self.last_epoch
        if step < self.warmup_steps:
            scale = step / max(1, self.warmup_steps)
        else:
            progress = (step - self.warmup_steps) / max(1, self.total_steps - self.warmup_steps)
            scale = max(0, 0.5 * (1 + math.cos(math.pi * progress)))
        return [max(self.min_lr, base_lr * scale) for base_lr in self.base_lrs]
Tip: Warmup 阶段一般设为总训练步数的 5%~10%，可以有效避免训练初期的梯度爆炸问题。
AI 编程 2026-02-12

混合精度训练：用 AMP 将训练速度提升 2 倍

PyTorch 的 Automatic Mixed Precision (AMP) 自动将部分运算降为 FP16，在几乎不损失精度的情况下大幅加速训练并降低显存占用。

from torch.cuda.amp import autocast, GradScaler

scaler = GradScaler()

for data, target in dataloader:
    optimizer.zero_grad()

    with autocast():
        output = model(data)
        loss = criterion(output, target)

    scaler.scale(loss).backward()
    scaler.step(optimizer)
    scaler.update()
Tip: 并非所有层都适合 FP16 — softmax、layer_norm 等数值敏感操作会被 AMP 自动保留为 FP32，无需手动干预。

Prompt 2026-02-15

Chain-of-Thought：用思维链提升推理准确率

对于需要逻辑推理的任务，直接问答的准确率往往不高。通过引导模型「一步步思考」，可以显著提升复杂问题的回答质量。

# 差的 prompt
"24 个人 3 天完成一个项目，18 个人需要几天？"

# 好的 prompt（思维链引导）
"24 个人 3 天完成一个项目，18 个人需要几天？
请一步步推理：
1. 先计算总工作量（人·天）
2. 再用总工作量除以新的人数
3. 给出最终答案"
Tip: 在 few-shot 示例中也展示推理过程（而非只给答案），效果会更好。这就是 Google 论文中提出的 Chain-of-Thought Prompting 的核心思路。

用 Worker Threads 实现 CPU 密集型任务并行

Node.js 的单线程模型在遇到 CPU 密集任务时会阻塞事件循环。worker_threads 模块可以创建真正的多线程并行计算。

import { Worker, isMainThread, parentPort, workerData } from 'worker_threads';

if (isMainThread) {
  const runWorker = (data) =>
    new Promise((resolve, reject) => {
      const worker = new Worker(new URL(import.meta.url), { workerData: data });
      worker.on('message', resolve);
      worker.on('error', reject);
    });

  const results = await Promise.all([
    runWorker({ start: 0, end: 5_000_000 }),
    runWorker({ start: 5_000_000, end: 10_000_000 }),
  ]);

  console.log('Total:', results.reduce((a, b) => a + b, 0));
} else {
  let sum = 0;
  for (let i = workerData.start; i < workerData.end; i++) sum += i;
  parentPort.postMessage(sum);
}
Tip: Worker 创建有开销，适合长时间运算的任务。对于短任务建议使用 worker pool（如 piscina 库）来复用线程。
Node.js 2026-02-10

用 AbortController 优雅取消异步操作

Node.js 原生支持 AbortController，可以统一取消 fetch 请求、定时器、流等异步操作，避免资源泄漏。

const controller = new AbortController();
const { signal } = controller;

// 5 秒超时自动取消
setTimeout(() => controller.abort(), 5000);

try {
  const res = await fetch('https://api.example.com/data', { signal });
  const data = await res.json();
  console.log(data);
} catch (err) {
  if (err.name === 'AbortError') {
    console.log('请求已取消（超时）');
  } else {
    throw err;
  }
}

// 也可用于 EventEmitter
import { on } from 'events';
for await (const event of on(emitter, 'data', { signal })) {
  process.stdout.write(event);
}
Tip: 从 Node.js 20 开始，AbortSignal.timeout(ms) 提供了更简洁的超时写法：fetch(url, { signal: AbortSignal.timeout(5000) })。
